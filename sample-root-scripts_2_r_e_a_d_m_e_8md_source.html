3 ## Inputs:
4 - the directory where the library (`libWCSimRoot.so`) resides is assumed to be in the above directory (`../libWCSimRoot.so`) unless the `$WCSIMDIR` environment variable, that defines the directory where the WCSim package is, is set.
5  - Alternatively (recommended) use the `rootwc` ROOT wrapper to automatically load all relevant libraries
6 - the input root file is assumed to be ../wcsim.root, unless it is given as argument to the macro.

8 ## Files:
9 - `sample_readfile.C`
10  - A general example macro to show how to access some (but not all) information stored in the output file
11  - See `$WCSIMDIR/include/WCSimRoot*.hh` for how to access other information that is not shown in this example
12  - Note: there are two arguments in `sample_readfile.C`, one is the root file and the other it's verbose (`true` or `false`). Verbose is `false` by default.
13 - `read_number_of_PMTs.C`
14  - Simply prints how many PMTs, of each type
15 - `read_PMT.C`
16 - `testgeo.C`
17 - `MakeKin.py`
18 - `hadd_wcsim_compileMe.C` and `hadd_wcsim_macro.C`
19  - Script to allow successful `hadd`-ing of WCSim files. The standard ROOT `hadd` executable cannot be used, due to the complicated `WCSimRoot*` class structures
20 - `AnalyzeWSHierarchy.c`
21 - `calcPhotoCoverage.C`
22  - Script to generate ID hybrid photocoverage options such that the *extra tower* is not built. Useful for reducing the number of overlaps/killed tracks in your geometry
23 - `KinSplitter.py`
24  - Take an input `.kin` file and combine all input vertices into multiple overlapping WCSim input events, based on the time of each input vertex
25  - Creates multiple `.kin` files per input `.kin` file
26  - Its main use is for simulating pileup of supernova events from sntools
27 - `MakeKin.py`
28  - Generate particle-gun like `.kin` files for WCSim to read
29  - As well as fixed position/direction, randomised position/directions are possible. A few options are implmented
30 - `nuPRISMconvert.cc`
31 - `plot_pmts.C`
32  - Plot all PMT positions. Useful for checking new geometries (e.g. if you've modified photocoverage options)
33 - `pmtremove.C`
34 - `read_OD.C`
35 - `trigger_OD.C`
