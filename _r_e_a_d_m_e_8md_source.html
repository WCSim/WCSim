64 ### Prerequisites
65 
66 The automated building testing is currently performed inside the hk-software docker container with the prerequisites:
67 * Geant4 10.3.3
68 * ROOT 6.26/04
69 * gcc 8.5.0
70 * cmake 3.20.2
71 
72 And as such this is the most supported version of the software (it is guaranteed to work)
73 
74 Other versions of prerequisite software can be tried, but aren't guaranteed to work.
75 * Geant 4.10.03p3
76  * If using a newer version of Geant, it is your responsiblity to assess whether Geant physics changes are significant, relative to the officially supported 4.10.03p3
77  * All of the G4 data files are also required to be installed, include hadron xsec, etc.
78 * ROOT
79  * ROOT5 (e.g. v5r34p36 or v5r34p38, or potentially any v5r34) can potentially be made to work. You may need to modify `$WCSIM_BUILD_DIR/src/cmake_install.cmake` to take account of ROOT5 not producing `.pcm` files
80  * ROOT6 has been tested on an older version than above (6.20/04)
81 * cmake 3.1 and above are required by `CMakeLists.txt`, but not all old versions have been tested
82 * The compiler should support `-std=c++11` (or `-std=c++0x`) e.g. gcc 4.8.5
83  * You may need to modify `CMakeLists.txt` to change the `CMAKE_CXX_STANDARD` to another version (e.g. from 14 to 17). I *think* this is dependent on what your prerequisties have been compiled with

85 ### Build Instructions using CMake:
86 
87 CMake is cross-platform software for managing the build process in
88 a compiler-independent way (cmake.org).
89 It is required to build ROOT and GEANT4 also through CMake.
90 **Cmake 3.1+ is required.**
91 Using cmake, builds and source code need to well separated and make
92 it easier to build many versions of the same software.
93 
94 A recommended way to set up the directory structure in your own
95 preferred `WCSIM_HOME`:
96 - `${WCSIM_HOME}/WCSim` : contains the src dir, typically the cloned or
97  unzipped code from GitHub
98 - `${WCSIM_HOME}/WCSim_build` : contains directories for each build, eg.
99  for each branch you want to test or for different releases, comparing
100  debugged versions, etc.
101  Each subdirectory like `WCSim_dev` or `WCSim_v1.2` would contain the intermediate files produced
102  during compilation and not the finale ones
103 - `${WCSIM_HOME}/WCSim_install` : contains directories for each build, eg.
104  for each branch you want to test or for different releases, comparing
105  debugged versions, etc.
106  Each subdirectory like `WCSim_dev` or `WCSim_v1.2` will contain the final executable and libraries,
107  the example macros and library for ROOT.
108 
109 To compile you need to have `CMakeLists.txt` in the WCSim source dir.
110 * `mkdir -p ${WCSIM_HOME}/WCSim_build/mydir ; cd ${WCSIM_HOME}/WCSim_build/mydir`
111 * Set up the environment variables
112  ```bash
113  #WCSim things
114  export WCSIMDIR=/${WCSIM_HOME}/WCSim
115  export G4WORKDIR=$WCSIMDIR/exe
116  #Geant4 things
117  export G4INSTALLDIR=/path/to/geant4/install/dir/
118  source $G4INSTALLDIR/bin/geant4.sh
119  source $G4INSTALLDIR/share/Geant4-10.3.3/geant4make/geant4make.sh
120  ```
121 * `cmake3 ../../WCSim` : this executes the commands in `CMakeLists.txt` and generates
122  the Makefiles for both the ROOT library as the main executable.
123  * For some OS, `cmake3` is just `cmake`
124 * `make clean` : if necessary
125 * `make -j3 install` : will first compile the libWCSimRoot.so and libWCSimCore.so which you need for using
126  the ROOT Dict from WCSim (e.g. to read the output files) and then compile WCSim.
127 * `source bin/this_wcsim.sh` : will setup the environment variables such that you can run WCSim
128 
129 To recompile:
130 * Typically just `make install` will be enough and also redo the cmake phase if
131  something changed.
132 * Sometimes you need to `make clean` first.
133 * When there are problems, try removing `CMakeCache.txt`, and redo the cmake.
134 
135 Useful cmake commands:
136 * `make edit_cache` : customize the build.
137 * `make rebuild_cache` : redo the cmake phase.
138 
139 #### WCSim cmake build options
140 * `-DWCSim_Geometry_Overlaps_CHECK=<ON|OFF>` If ON, turns on geometry overlap checking (slow, but important when setting new detector geometry options). Default: OFF
141 * `-DWCSim_DEBUG_COMPILE_FLAG=<ON|OFF>` If ON, turns on the gcc debug compiler flag `-g`. Default: OFF
142 * `-DWCSIM_SAVE_PHOTON_HISTORY_FLAG=<ON|OFF>` If ON, turns on photon scattering/reflection history saving. The data class `WCSimRootCherenkovHitHistory` is used in a similar way as `WCSimRootCherenkovHitTime`. Default: OFF
143 
144 #### Build with CMake on sukap:
145 
146 A script allowing to build WCSim with CMake on sukap is available:
147 ```bash
148 cd $WCSIMDIR
149 ./setup/env_sukap.sh
150 ./make.sh
151 ```
152 Note that `./make.sh` should also work on other systems.
153 
154 This script will read the current WCSim directory name and create a directory to hold the different cmake builds.
155 For example, in case your current WCSim directory name is 'WCSim', it will create a '../WCSim-build' directory. It will also create a `../WCSim-install` directory.
156 
157 It will then create a directory for your current branch which will have the name of the current branch.
158 For example, in case your current branch name is `hybridPMT`, you will have:
159 ```bash
160 WCSim/
161 WCSim-build/
162 WCSim-build/hybridPMT/
163 WCSim-install/
164 WCSim-install/hybridPMT/
165 ```
166 
167 Note that by defaul, just `make install` is run.
168 You can run `make clean` before `make install` by running `./make.sh clean`

178 ### Using WCSim without building using Docker:
179 
180 Docker allows you to use WCSim without compiling in an OS independant way. The Docker images are hosted on DockerHub and can be used by following the steps below.
181 
182 1) Install Docker cross platform instructions can be found at https://www.docker.com/
183 2) Pull the WCSim image from docker hub by using `docker pull wcsim/wcsim:tag` where `tag` is the tagged version or use the tag `latest` to get the current develop branch
184 3) Run the docker image and create a container `docker run --name=WCSim -i -t wcsim/wcsim:tag` this will give you a shell in the container's OS with WCSim already built.
185 To save data from inside your docker image mount a local folder in the docker image at runtime and then anything placed in that directory will be available in that folder after exit. To do that run the following `docker run -v local_folder_path:docker_mount_path -i -t wcsim/wcsim:tag`
186 4) Once you have run the docker image, you will already be in `$WCSIMDIR` and WCSim (and prerequisites) will be setup. Therefore, you can just start running WCSim as normal from this directory
187 5) To exit the docker image `exit`
188 
189 (Note: You only need to use the `docker run` command once to create the container. Once created you changes are saved in that container instance and you can start and stop the container at any time with `docker start WCSim` and `docker stop WCSim`);
190 
191 Extra docker commands:
192 1) See all images `docker images`
193 2) Delete an image `docker rmi imageID`
194 3) See all containers `docker ps -a`
195 4) Delete a container `docker rm ContainerID`
196 
197 Note that this method currently uses make (not cmake) to build

199 ### Using WCSim without building using Singularity
200 
201 Singularity is a similar container tool with different philosphies. The most important being that you can't run as root. This means that it may be installed and available to use on your local cluster.
202 
203 You should be able to run the docker container with singularity without any problems. Just to note that `$WCSIMDIR` will be read-only, therefore you should run WCSim elsewhere (if you forget you'll see a nasty seg fault - this is just because of the read-only directory).

207 ## Running WCSim
208 
209 To test that WCSim is working, try running the test macro `WCSim.mac`, which runs 10 electrons with 5 MeV of energy in the Super-Kamiokande detector (it should take less than a minute to run).
210 
211 ```bash
212 cd /path/to/wcsim/build/dir/
213 ./WCSim WCSim.mac macros/tuning_parameters.mac
214 ```
215 
216 `WCSim.mac` is well commented. Take a look inside (and also at other `.mac` files in `/macros/`) for the various options you can use to run WCSim
