36 ## Current notes and how to build
37 
38 You should have a recent and working version of ROOT and GEANT4.
39 (Known to work with GEANT 4.10.1p03 and ROOT v5.28.00) You also need all of the G4
40 data files including hadron xsecs etc. Those are the only
41 requirements. The code should work with gcc 4.4.7. For v1.6.0 and earlier, use GEANT 4.9.4.p01.
42 
43 ### Build Instructions using make:
44 
45 To compile:
46 * `make clean`
47 * `make rootcint`
48 * `make`
49 
50 If you want to use these libraries with an external program then also do:
51 * `make shared` [ For root programs]
52 * `make libWCSim.a` [ Also necessary for the event display?]
53 
54 ### Build Instructions using CMake:
55 
56 CMake is cross-platform software for managing the build process in
57 a compiler-independent way (cmake.org).
58 It is recommended to build ROOT and GEANT4 also through CMake. The
59 latter is very CMake friendly since GEANT 4.9.6, while it started introducing
60 builds through CMake from 4.9.4 onwards (http://geant4.web.cern.ch/geant4/support/ReleaseNotes4.9.4.html#10.).
61 Using cmake, builds and source code need to well separated and make
62 it easier to build many versions of the same software.
63 
64 A recommended way to set up the directory structure in your own
65 preferred WCSIM_HOME:
66 - `${WCSIM_HOME}/WCSim` : contains the src dir, typically the cloned or
67  unzipped code from GitHub
68 - `${WCSIM_HOME}/WCSim_build` : contains directories for each build, eg.
69  for each branch you want to test or for different releases, comparing
70  debugged versions, etc.
71  This directory will contain the executable, the example macros and
72  library for ROOT.
73 
74 To compile you need to have `CMakeLists.txt` in the WCSim source dir.
75 * `mkdir ${WCSIM_HOME}/WCSim_build/mydir ; cd ${WCSIM_HOME}/WCSim_build/mydir`
76 * Set up the Geant4_Dir: `export Geant4_DIR=${HOME}/Geant4/install/geant4.9.6.p04`
77  (from the make install phase of Geant4)
78 * `cmake ../../WCSim` : this executes the commands in `CMakeLists.txt` and generates
79  the Makefiles for both the ROOT library as the main executable.
80 * `make clean` : if necessary
81 * `make` : will first compile the libWCSimRoot.so which you need for using
82  the ROOT Dict from WCSim and then compile WCSim.
83 
84 To recompile:
85 * Typically just `make` will be enough and also redo the cmake phase if
86  something changed.
87 * Sometimes you need to `make clean` first.
88 * When there are problems, try removing `CMakeCache.txt`, and redo the cmake.
89 
90 Useful cmake commands:
91 * `make edit_cache` : customize the build.
92 * `make rebuild_cache` : redo the cmake phase.

95 ### Using WCSim without building using Docker:
96 
97 Docker allows you to use WCSim without compiling in an OS independant way. The Docker images are hosted on DockerHub and can be used by following the steps below.
98 
99 1) Install Docker cross platform instructions can be found at https://www.docker.com/
100 2) Pull the WCSim image from docker hub by using `docker pull wcsim/wcsim:tag` where `tag` is the tagged version or use the tag `latest` to get the current develop branch
101 3) Run the docker image and create a container `docker run --name=WCSim -i -t wcsim/wcsim:tag` this will give you a shell in the container's OS with WCSim already built.
102 To save data from inside your docker image mount a local folder in the docker image at runtime and then anything placed in that directory will be available in that folder after exit. To do that run the following `docker run -v local_folder_path:docker_mount_path -i -t wcsim/wcsim:tag`
103 4) Once you have run the docker image, you will already be in `$WCSIMDIR` and WCSim (and prerequisites) will be setup. Therefore, you can just start running WCSim as normal from this directory
104 5) To exit the docker image `exit`
105 
106 (Note: You only need to use the `docker run` command once to create the container. Once created you changes are saved in that container instance and you can start and stop the contianer at any time with `docker start WCSim` and `docker stop WCSim`);
107 
108 Extra docker commands:
109 1) See all images `docker images`
110 2) Delete an image `docker rmi imageID`
111 3) See all containers `docker ps -a`
112 4) Delete a container `docker rm ContainerID`

114 #### Using WCSim without building using Singularity
115 
116 Singularity is a similar container tool with different philosphies. The most important being that you can't run as root. This means that it may be installed and available to use on your local cluster.
117 
118 You should be able to run the docker container with singularity without any problems. Just to note that $WCSIMDIR will be read-only, therefore you should run WCSim elsewhere (if you forget you'll see a nasty seg fault - this is just because of the read-only directory).

120 ## Running WCSim
121 
122 To test that WCSim is working, try running the test macro `WCSim.mac`, which runs 10 electrons with 500 MeV of energy in the Super-Kamiokande detector. The command is one of the following, depending on how WCSim was built:
123 
124 `./bin/Linux-g++/WCSim WCSim.mac`
125 
126 `./exe/bin/Linux-g++/WCSim WCSim.mac`
127 
128 `WCSim.mac` is well commented. Take a look inside (and also at other `.mac` files in `/macros/`) for the various options you can use to run WCSim
